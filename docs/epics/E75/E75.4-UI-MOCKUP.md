## Responsive Behavior

The component is designed to work on:
- Desktop: Full width with side-by-side action buttons
- Tablet: Stacked layout with full-width buttons
- Mobile: Single column, touch-friendly buttons

## Accessibility Features

- Proper ARIA labels on all interactive elements
- Keyboard navigation support (Tab, Enter, Escape)
- Focus management in modals
- Screen reader compatible
- High contrast support (dark mode)

## Color Scheme

- **Primary**: Blue (#0ea5e9) - Primary actions, links
- **Success**: Green (#10b981) - Success states
- **Danger**: Red (#ef4444) - Delete/Archive, errors
- **Warning**: Amber (#f59e0b) - Warnings
- **Secondary**: Slate - Text, borders, backgrounds
- **Entry Type Badges**: Secondary variant (slate)
- **Version Badge**: Secondary variant with clock icon

## Entry Type Color Coding (Future Enhancement)

While currently all using secondary badge variant, entry types could be color-coded:
- **Krankengeschichte** (Medical History): Blue
- **Symptome** (Symptoms): Orange
- **Medikation** (Medications): Purple
- **Allergien** (Allergies): Red
- **Familienanamnese** (Family History): Green
- **Lebensstil** (Lifestyle): Teal
- **Sonstiges** (Other): Gray
