# Rhythmologicum Connect – Version 0.4 Plan

Kurzfassung: Dies ist der kompakte v0.4-Plan. Für die ausführliche GitHub-Planung siehe `V0_4_FULL_PLAN_GITHUB.md`.

## Vision

v0.4 liefert eine moderne, “live-ready” Web-Version mit:

- Globalem UI-Refresh & Design System
- Patient Flow V2 (mobilfreundlich, konsistent)
- Content Flow Engine (CONTENT_PAGE-Integration)
- Clinician Dashboard V2
- Role-based Navigation & Routing
- Technische Bereinigung, Stabilität und bessere DX

## Executive Summary

Version 0.4 aims to deliver the first “live-ready” web-based version of Rhythmologicum Connect. Focus areas include:

- Modern UI/UX redesign (Clinician/Admin + global patient experience)
- Patient Flow V2 (clean, consistent journey, mobile-friendly)
- Content Flow Integration (CONTENT_PAGE nodes within funnels)
- Improved navigation, routing, and role handling
- Preparation for external test patients
- Technical consistency, cleanup, and improved developer experience

## Core Epics (v0.4)

1. **UI Refresh & Design System**
2. **Patient Flow V2**
3. **Content Flow Engine (CONTENT_PAGE nodes)**
4. **Clinician Dashboard Redesign**
5. **Navigation & Access Control Overhaul**
6. **Funnel Engine V2 (Extensible Architecture)**
7. **Performance, Error Handling, and Stability**

## Architecture Overview

- Centralized Layout Components
- Design Tokens + Tailwind Themes
- Funnel Engine: Modular Node Types (QUESTION, RESULT, CONTENT_PAGE)
- API Layer Clean-Up & Standardization
- Role-based routing via middleware
- # Unified Data Fetching Strategy
