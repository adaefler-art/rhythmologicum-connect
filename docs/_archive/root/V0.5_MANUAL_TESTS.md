# v0.5 Manual Tests (P0)

**Scope:** Evidence-first, step-by-step manual verification for v0.5.

**Rules:**
- Every step either references a file path (implementation surface) or says “verify via command”.
- Run locally with `npm run dev` (verify via command) or against the deployed environment.

---

## P0 Patient journey

1) Sign in as a patient user
- Entry page: `app/page.tsx`
- Auth callback endpoint: `app/api/auth/callback/route.ts`
- Verify: you can reach `/patient` after sign-in.

2) Open patient home
- Page: `app/patient/page.tsx`
- Verify: patient navigation renders and no access error is shown.

3) Complete onboarding consent
- Page: `app/patient/onboarding/consent/page.tsx`
- Verify: you can proceed to the next onboarding step.

4) Complete onboarding profile
- Page: `app/patient/onboarding/profile/page.tsx`
- Verify: you can save profile and continue.

5) Open funnel catalog
- Page: `app/patient/funnels/page.tsx`
- Verify: you can select a funnel.

6) Start a funnel and reach the intro
- Intro page: `app/patient/funnel/[slug]/intro/page.tsx`
- Verify: the intro renders for the selected slug.

7) Proceed through funnel content and questions
- Funnel step page: `app/patient/funnel/[slug]/page.tsx`
- Content page: `app/patient/funnel/[slug]/content/[pageSlug]/page.tsx`
- Verify: you can advance without client errors.

8) Reach funnel result
- Result page: `app/patient/funnel/[slug]/result/page.tsx`
- Verify: result renders after completion.

9) Verify history is accessible
- Page: `app/patient/history/page.tsx`
- Verify: history page loads.

---

## P0 Clinician journey

1) Sign in as a clinician user
- Clinician layout gate (client-side): `app/clinician/layout.tsx`
- Verify: you can reach `/clinician`.

2) Open clinician home
- Page: `app/clinician/page.tsx`
- Verify: clinician dashboard loads.

3) Open review queue
- Page: `app/clinician/review-queue/page.tsx`
- Backend endpoint: `app/api/review/queue/route.ts`
- Verify: the queue loads (no 401/403) and items render.

4) Open a review item details (if available)
- API surface (verify via file paths):
  - `app/api/review/[id]/route.ts`
  - `app/api/review/[id]/details/route.ts`
  - `app/api/review/[id]/decide/route.ts`
- Verify: you can view details and submit a decision.

5) Open a patient detail page (requires a valid patient id)
- Page: `app/clinician/patient/[id]/page.tsx`
- Verify: patient sections render for a known patient.

---

## Evidence capture

- For each failure, capture the terminal output (if local) and the full request/response in browser devtools.
- For build readiness, verify via commands listed in `RELEASE_CHECKLIST_V0.5.md`.
